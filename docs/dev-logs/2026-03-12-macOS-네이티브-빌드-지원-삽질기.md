# 멀티플랫폼 앱에 macOS 빌드 추가하면서 만난 에러 6종 세트

## 상황

Writ은 처음부터 iPhone/iPad/Mac/Apple Watch 멀티플랫폼으로 설계된 앱이다. 그런데 project.yml의 `SUPPORTED_PLATFORMS`에 `macosx`가 빠져 있어서 Xcode에서 "My Mac"을 선택하면 "Designed for iPad"로만 뜨는 상태였다. `SUPPORTED_PLATFORMS`에 `macosx`를 추가해 네이티브 macOS 빌드를 시도했더니 에러가 줄줄이 터졌다.

## 에러 내용

### 1. WatchConnectivity 모듈 없음

```
PhoneWatchSessionManager.swift:2:8: error: Unable to find module dependency: 'WatchConnectivity'
```

### 2. iOS 전용 SwiftUI API

```
RecordingView.swift:75:14: error: 'navigationBarTitleDisplayMode' is unavailable in macOS
RecordingView.swift:76:66: error: 'navigationBar' is unavailable in macOS
HistoryView.swift:77:21: error: 'insetGrouped' is unavailable in macOS
TranscriptionDetailView.swift:58:10: error: 'navigationBarTitleDisplayMode' is unavailable in macOS
RetranscribeSheet.swift:41:14: error: 'navigationBarTitleDisplayMode' is unavailable in macOS
```

### 3. ActivityKit macOS 미지원

```
WritActivityAttributes.swift:4:32: error: 'ActivityAttributes' is unavailable in macOS
```

### 4. iOS extension이 macOS 빌드에 포함

```
error: Your target is built for macOS but contains embedded content built for the iOS platform (WritKeyboard.appex), which is not allowed.
error: Your target is built for macOS but contains embedded content built for the iOS platform (WritWidgetExtension.appex), which is not allowed.
```

### 5. XcodeGen이 entitlements 파일을 초기화

xcodegen generate를 실행할 때마다 `Writ.entitlements`와 `WritKeyboard.entitlements`가 빈 `<dict/>`로 리셋되는 현상.

## 원인 분석

**1번**: `WatchConnectivity`는 iOS/watchOS 전용 프레임워크. macOS SDK에 존재하지 않는다.

**2번**: `navigationBarTitleDisplayMode`, `toolbarColorScheme(_:for: .navigationBar)`, `.listStyle(.insetGrouped)`는 iOS에만 존재하는 SwiftUI modifier. macOS에서는 NavigationBar 개념 자체가 다르다.

**3번**: ActivityKit(Live Activity)은 iOS 16.1+에서 도입됐고, macOS에서는 아직 네이티브로 사용 불가. macOS 26에서 Live Activity가 보이는 건 **iPhone 미러링**이지 macOS 앱이 직접 생성하는 게 아니다. 처음에 `canImport(ActivityKit)`으로 가드했는데 macOS에서도 true를 반환해서 의미가 없었다.

**4번**: 멀티플랫폼 타겟에서 iOS 전용 extension(키보드, 위젯)이 macOS 빌드에도 embed되려고 해서 발생. XcodeGen의 `platforms: [iOS]` 옵션이 pbxproj에 `platformFilter`를 생성하지 못하는 버그(또는 미지원).

**5번**: XcodeGen의 `entitlements: path:` 옵션만 사용하면, generate 시 해당 파일을 빈 plist로 덮어쓴다.

## 해결 과정

**1번 WatchConnectivity**: 파일 전체를 `#if os(iOS)` / `#endif`로 감싸고, AppState에서 참조하는 부분도 동일하게 가드.

```swift
// PhoneWatchSessionManager.swift
#if os(iOS)
import WatchConnectivity
// ... 전체 클래스
#endif
```

**2번 iOS 전용 API**: 각 사용처에 `#if os(iOS)` 가드 추가.

```swift
.navigationTitle("녹음")
#if os(iOS)
.navigationBarTitleDisplayMode(.large)
.toolbarColorScheme(isRecording ? .dark : nil, for: .navigationBar)
#endif
```

**3번 ActivityKit**: `#if canImport(ActivityKit)` → `#if os(iOS)`로 변경. `canImport`는 macOS에서도 true를 반환하므로 의미 없음.

```swift
// WritActivityAttributes.swift
#if os(iOS)
import ActivityKit
struct WritActivityAttributes: ActivityAttributes { ... }
#endif
```

**4번 Extension embed**: XcodeGen이 `platforms: [iOS]`를 pbxproj `platformFilter`로 변환하지 못해서, xcodegen generate 후 Python 스크립트로 pbxproj를 직접 패치.

```python
# PBXTargetDependency에 platformFilter = ios 추가
content = re.sub(
    r'(isa = PBXTargetDependency;\n\t\t\ttarget =)',
    r'isa = PBXTargetDependency;\n\t\t\tplatformFilter = ios;\n\t\t\ttarget =',
    content
)
# PBXBuildFile embed 항목에도 platformFilter = ios 추가
```

**5번 Entitlements 초기화**: `entitlements:` 블록에 `properties:`를 명시하면 xcodegen이 값을 포함한 파일을 생성한다.

```yaml
entitlements:
  path: Writ/Writ.entitlements
  properties:
    com.apple.security.application-groups:
      - group.com.solstice.writ
```

## 배운 점

멀티플랫폼 앱에서는 **`#if os(iOS)`가 `#if canImport`보다 신뢰할 수 있다** — canImport는 SDK에 헤더가 있으면 true를 반환하므로 "사용 가능"과 "import 가능"이 다를 수 있다. 그리고 XcodeGen은 멀티플랫폼 extension dependency 필터링을 제대로 지원하지 못하므로, pbxproj 후처리가 필요할 수 있다.
