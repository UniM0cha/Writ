# MLX GPU 메모리 미해제로 인한 모델 전환/연속 전사 OOM 크래시

## 상황

iPhone 14 Pro(6GB)에서 Qwen3-ASR 엔진 사용 시 두 가지 OOM 크래시 발생:
1. 0.6B 8-bit 모델 사용 후 0.6B 4-bit로 전환하면 크래시
2. 동일 모델로 첫 번째 전사 후 두 번째 전사 시작 시 크래시

## 에러 내용

iOS jetsam에 의한 강제 종료 (메모리 초과). Xcode에서 "Message from debugger: Terminated due to memory issue" 표시.

## 원인 분석

3가지 원인이 복합적으로 작용:

### 1. ForcedAligner 메모리 미해제

`Qwen3ASREngine.unloadModel()`에서 ASR 모델은 `model.unload()` → `clearParameters()`로 GPU 메모리를 해제하지만, ForcedAligner는 `self.aligner = nil`만 설정하여 ARC에 의존했다.

```swift
// 수정 전: Qwen3ASREngine.swift
func unloadModel() async {
    let modelToUnload = lock.withLock { ... }
    modelToUnload?.unload()  // ASR만 해제
    // aligner는 nil 설정만 — MLX GPU 메모리 잔류
}
```

`Qwen3ForcedAligner`는 `ModelMemoryManageable` 프로토콜을 구현하지 않아 `unload()` 메서드가 없다. 내부의 `audioEncoder`, `textDecoder`, `classifyHead` 모두 MLX `Module`이므로 `clearParameters()`를 직접 호출해야 GPU 메모리가 해제된다.

### 2. WhisperKit 모델 미해제

`WhisperKitEngine.unloadModel()`에서 `self.whisperKit = nil`만 설정하고, WhisperKit의 `unloadModels()`를 호출하지 않았다. CoreML 모델의 메모리가 ARC 타이밍에 의존하여 즉시 해제되지 않았다.

### 3. MLX 캐시 무제한 증가

MLX는 성능 최적화를 위해 해제된 텐서의 GPU 버퍼를 캐시 풀에 보관한다. `Memory.cacheLimit`을 설정하지 않으면 캐시가 무한히 커질 수 있다. 전사 시 생성되는 KV cache 등 중간 텐서가 함수 종료 후에도 GPU 메모리 풀에 남아 있었다.

## 해결 과정

### ForcedAligner 명시적 메모리 해제

```diff
 // Qwen3ASREngine.swift
 func unloadModel() async {
-    let modelToUnload = lock.withLock { () -> Qwen3ASRModel? in
+    let (modelToUnload, alignerToUnload) = lock.withLock { () -> (Qwen3ASRModel?, Qwen3ForcedAligner?) in
         let m = self.model
+        let a = self.aligner
         self.model = nil
         self.aligner = nil
         self._currentModel = nil
-        return m
+        return (m, a)
     }
     modelToUnload?.unload()
+    // ForcedAligner는 ModelMemoryManageable 미구현 → 직접 clearParameters
+    alignerToUnload?.audioEncoder.clearParameters()
+    (alignerToUnload?.textDecoder as? Module)?.clearParameters()
+    alignerToUnload?.classifyHead.clearParameters()
 }
```

`clearParameters()`는 MLX `Module` extension(`MLXCommon/ModuleMemory.swift`)에 정의되어 있고, 내부적으로 모든 파라미터를 빈 `MLXArray()`로 교체한 뒤 `Memory.clearCache()`까지 호출한다.

### WhisperKit 명시적 모델 해제

```diff
 // WhisperKitEngine.swift
 func unloadModel() async {
-    lock.withLock {
-        self.whisperKit = nil
-        self._currentModel = nil
+    let kit = lock.withLock { () -> WhisperKit? in
+        let k = self.whisperKit
+        self.whisperKit = nil
+        self._currentModel = nil
+        return k
     }
+    await kit?.unloadModels()
 }
```

### MLX 캐시 제한 설정

```swift
// Qwen3ASREngine init()
init() {
    Memory.cacheLimit = 64 * 1024 * 1024 // 64MB
}
```

## 배운 점

MLX는 서버/데스크톱을 위해 설계된 프레임워크라 GPU 메모리를 적극적으로 캐시한다. iOS처럼 메모리가 제한된 환경에서는 반드시 `Memory.cacheLimit`을 설정하고, 모델 해제 시 `clearParameters()`를 명시적으로 호출해야 한다. ARC의 `nil` 설정만으로는 MLX GPU 메모리가 즉시 반환되지 않는다.
